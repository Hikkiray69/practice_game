# STAGE 0 — Рамки проекта

## 0.1 Формат игры

**Решение:** одиночная интерактивная 3D-игра (single-player) с миссиями, NPC-диалогами и системой последствий выбора.

**Почему так:**
- лучше контролируется MVP-объем;
- проще стабилизировать геймплей и архитектуру;
- полностью покрывает требования ТЗ по обучению ценностям.

## 0.2 Платформа MVP

**Решение:** desktop-first веб-приложение с базовой адаптивностью под планшет/мобильные экраны.

**Минимальные целевые параметры:**
- Desktop: от `1280x720` и выше (приоритет).
- Tablet: от `768x1024` (базовая поддержка).
- Mobile: базовая читаемость UI и доступ к основному флоу без гарантии максимального комфорта управления в 3D.

## 0.3 Definition of Done (MVP-1)

MVP-1 (вертикальный срез) считается готовым, если выполнены все условия ниже:

### Геймплей
- есть 1 полноценная миссия (ценность "Ответственность");
- работает полный цикл: миссия -> взаимодействие -> выбор -> результат -> обновление прогресса.

### Технологии
- приложение реализовано на Next.js + TypeScript;
- 3D-сцена реализована на Three.js/react-three-fiber;
- применена FSD-структура (`app`, `processes`, `pages`, `features`, `entities`, `shared`);
- Cursor AI используется для генерации диалогов/подсказок (допускается fallback на статический контент).

### UX/UI
- есть HUD с минимумом: активная миссия, очки/прогресс, статус;
- есть понятные визуальные подсказки;
- базовая адаптивность сохранена.

### Качество
- нет блокирующих багов в основном сценарии;
- миссия полностью проходима от начала до конца;
- логика не нарушает границы слоев FSD.

### Документация
- README содержит: запуск, структуру FSD, описание механик, описание AI-интеграции.

## 0.4 Definition of Done (MVP-2, версия под ТЗ)

MVP-2 считается готовым, если:

- добавлены минимум 3 миссии;
- миссии покрывают ключевые ценности/принципы из ТЗ;
- сохранены стабильность, производительность и качество UX.

## 0.5 Риски этапа и контроль

- **Риск:** переразмеренный MVP.  
  **Контроль:** сначала делаем 1 миссию (MVP-1), затем масштабируем до 3 (MVP-2).

- **Риск:** перегрузка AI-части.  
  **Контроль:** сначала статические сценарии, затем AI-вариативность.

- **Риск:** просадка FPS в 3D.  
  **Контроль:** минималистичная сцена и легкие ассеты на MVP.
