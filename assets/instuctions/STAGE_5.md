# STAGE 5 — Реализация MVP-1 (основной код)

## 5.1 Цель этапа

Реализовать рабочий вертикальный срез `MVP-1` для первой миссии ("Ответственность") с подтвержденной работоспособностью в браузере.

## 5.2 Что реализовано в проекте

### Квестовый движок

- реализованы статусы миссии: `new`, `inProgress`, `completed`, `failed`;
- реализованы переходы состояний (старт миссии, выбор решения, завершение);
- реализована логика последствий выбора (изменение метрик).

Ключевой файл:
- `game/src/features/quest-system/model/useMission.ts`

### Диалоговая система

- реализована панель миссии с текстом контекста;
- реализован выбор из нескольких вариантов решения;
- после выбора отображается итог решения;
- добавлен retry-сценарий по принципу "право на ошибку".

Ключевой файл:
- `game/src/features/dialogue/ui/DialoguePanel.tsx`

### HUD и прогресс

- отображаются статус миссии, очки, скорость и качество;
- метрики меняются в реальном времени после выбора;
- применены `shared/constants` и `shared/lib` для форматирования и подписей.

Ключевые файлы:
- `game/src/features/scoring-progress/ui/Hud.tsx`
- `game/src/shared/constants/hud.ts`
- `game/src/shared/lib/formatMetric.ts`

### Перемещение игрока

- добавлено управление игроком (`WASD` + стрелки);
- движение ограничено пределами сцены.

Ключевой файл:
- `game/src/features/movement/ui/PlayerAvatar.tsx`

### Взаимодействие с NPC/объектами

- интеракция с NPC реализована через объект в сцене (куб);
- диалог доступен после взаимодействия с NPC;
- взаимодействие привязано к дистанции: нужно подойти достаточно близко;
- в сцену добавлены визуальные подсказки о текущем состоянии взаимодействия.

Ключевые файлы:
- `game/src/shared/ui/SceneCanvas.tsx`
- `game/src/app/game-screen/ui/GameScreen.tsx`

### Структурные улучшения FSD

- добавлены public API (`index.ts`) для слайсов и слоев;
- импорты переведены с deep-imports на публичные точки входа.

Ключевые файлы:
- `game/src/entities/quest/index.ts`
- `game/src/features/quest-system/index.ts`
- `game/src/features/dialogue/index.ts`
- `game/src/features/scoring-progress/index.ts`
- `game/src/features/movement/index.ts`
- `game/src/shared/ui/index.ts`

## 5.3 Проверка работоспособности

Технические проверки:

- `npm run lint` — успешно;
- `npm run build` — успешно.

Функциональные проверки (ручные):

- 3D-сцена отображается корректно;
- HUD отображает актуальные метрики;
- после старта миссии доступны варианты выбора;
- выбор влияет на очки/качество/скорость;
- статус миссии меняется корректно;
- retry возвращает миссию в рабочее состояние;
- перемещение и взаимодействие с NPC работают.

## 5.4 Что остается на следующие этапы

Для `MVP-2` и последующих этапов остаются:

- финальная регрессионная проверка после масштабирования до 3 миссий;
- расширенная AI-интеграция (генерация диалогов/подсказок);
- таймер-челлендж как полноценная механика;
- отдельные маршруты `main-menu`, `mission-result`, `profile/progress` (при необходимости выноса из единого экрана).

## 5.6 Обновление под MVP-2 (этап 5B, частично)

В коде реализованы:

- миссия 2 (`Прозрачность`) и миссия 3 (`Скорость`) как `QuestScenario`;

- суммарные метрики прогресса (`score`, `quality`, `speed`) в HUD;
- совместимость с механиками миссии 1 без регрессии сборки.

Ключевые файлы:

- `game/src/features/quest-system/model/useMission.ts`
- `game/src/features/dialogue/ui/DialoguePanel.tsx`
- `game/src/features/scoring-progress/ui/Hud.tsx`
- `game/src/app/game-screen/ui/GameScreen.tsx`

Техническая проверка:

- `npm run lint` — успешно;
- `npm run build` — успешно.

Регрессионная проверка после масштабирования:

- подтверждено прохождение миссий 1 -> 2 -> 3 без поломки переходов;
- проверено обновление статусов и суммарных метрик на каждом шаге;
- retry-сценарий сохраняет корректную логику после добавления новых миссий.

## 5.5 Результат этапа

Этап 5 завершен:

- `MVP-1`: реализован и подтвержден (миссия 1 end-to-end);
- `MVP-2 (5A + 5B)`: миссии 2 и 3 спроектированы, реализованы и проверены.
