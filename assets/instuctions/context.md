# AI Context — Interactive Learning Game

## 1. Введение

Задача: создать интерактивную обучающую игру, которая в игровой форме знакомит новых сотрудников и пользователей с ценностями и принципами компании.

Технологический стек:
- Next.js
- Three.js (или react-three-fiber)
- Cursor AI
- Feature-Sliced Design (FSD)

## 2. Цели проекта

- **Обучающая цель:** донести ценности и принципы компании через игровой опыт.
- **Техническая цель:** показать владение Next.js, Three.js, Cursor AI и FSD.
- **Продуктовая цель:** повысить вовлеченность за счет сюжета и игровых механик.

## 3. Ценности компании

1. **Ответственность**
   - Основа отношений — ответственность за результат.
   - Ответственность не ограничивается формальными инструкциями.
2. **Прозрачность**
   - Открытое обсуждение проблем и решений внутри команды и с клиентами.
   - Ясность и доверие на каждом этапе проекта.
3. **Скорость**
   - Скорость достижения результата важна не меньше, чем качество.

## 4. Принципы работы

1. **Право на ошибку** — безопасно пробовать новое, извлекать уроки и двигаться дальше.
2. **Предпринимательский подход** — относиться к задачам как к собственному бизнесу.
3. **Открытое общение** — открыто обсуждать проблемы/решения и держать клиента в курсе.
4. **Развитие** — искать улучшения качества и ускорять выполнение задач.

## 5. Архитектура FSD

Проект должен быть организован по слоям:

```text
src/
  app/        # глобальные настройки, провайдеры, роутинг
  processes/  # сквозные бизнес-процессы приложения
  pages/      # страницы верхнего уровня
  features/   # отдельные пользовательские возможности
  entities/   # доменные сущности
  shared/     # общие модули, UI, утилиты, константы
```

Правила:
- разделять бизнес-логику, презентационную логику и инфраструктуру;
- избегать смешивания слоев;
- проектировать под масштабирование.

## 6. Описание игры и механики

1. **Игровая среда:** 3D-пространство на Three.js с перемещением и взаимодействием.
2. **Миссии и квесты:** каждая миссия отражает ценность или принцип.
3. **Принципы в механиках:**
   - право на ошибку -> безопасный эксперимент;
   - предпринимательский подход -> бизнес-решения и ответственность;
   - открытое общение -> диалоги и обмен информацией;
   - развитие -> прокачка навыков, ускорение.
4. **Геймификация:** очки, уровни, достижения.
5. **Прозрачность:** прогресс и результаты в реальном времени.
6. **Скорость:** таймеры/ограничения времени с сохранением качества.

## 7. Требования к реализации

- Next.js: SSR, роутинг, API Routes (при необходимости).
- Three.js: рендер и анимация 3D-сцены, оптимизация.
- Cursor AI: генерация диалогов, квестов, подсказок.
- FSD: строгая слоистая архитектура.
- Игровая логика: минимум 3 миссии.
- UI/UX: понятная навигация, визуальные подсказки, адаптивность.
- Документация: README (запуск, структура, применение FSD).

## 8. Критерии оценки

1. Логичность сюжета и привязка к ценностям/принципам.
2. Техническая реализация (чистота кода, FSD, оптимизация).
3. Игровая механика и UI/UX.
4. Интеграция и функциональность AI.
5. Оригинальность подхода.
6. Полнота документации.
